# 训练参数调优指南

## 概述

本指南提供训练参数调优的建议和最佳实践，帮助您获得更好的模型性能。

## 学习率调优

### 初始学习率选择

| 模型 | 推荐初始学习率 |
|------|----------------|
| ResNet-6/10/14 | 0.0003 - 0.001 |
| ResNet-18/34 | 0.0001 - 0.0003 |
| ResNet-50 | 0.0001 - 0.0002 |

### 学习率范围测试

使用小批次数据进行学习率范围测试，找到合适的学习率区间：

1. 从很小的学习率开始（如 1e-7）
2. 逐步增加学习率
3. 观察损失变化
4. 选择损失下降最快的区间

### Warmup 策略

- **warmup_ratio**: 通常设置为 0.05 - 0.1（5%-10% 的训练步数）
- **warmup_steps**: 如果已知总步数，可以直接设置
- **目的**: 避免训练初期梯度爆炸

## 批次大小调优

### 推荐批次大小

| GPU 显存 | 推荐批次大小 |
|----------|--------------|
| 4GB - 6GB | 16 - 32 |
| 8GB - 12GB | 32 - 64 |
| 12GB+ | 64 - 128 |

### 批次大小与学习率的关系

当改变批次大小时，通常需要相应调整学习率：

```
新学习率 = 旧学习率 × sqrt(新批次大小 / 旧批次大小)
```

### 累积梯度

如果 GPU 显存不足以支持大批次，可以使用梯度累积（需要修改代码）。

## 训练轮数

### 推荐训练轮数

| 数据集大小 | 推荐训练轮数 |
|------------|--------------|
| 小型数据集 (< 1k) | 100 - 200 |
| 中型数据集 (1k - 10k) | 50 - 100 |
| 大型数据集 (> 10k) | 30 - 60 |

### 早停策略

观察验证集准确率，当连续多个 epoch 不再提升时，可以提前停止训练。

## 正则化参数调优

### Dropout

| 过拟合程度 | 推荐 dropout_p |
|------------|----------------|
| 无过拟合 | 0.0 - 0.2 |
| 轻微过拟合 | 0.2 - 0.4 |
| 严重过拟合 | 0.4 - 0.6 |

### 权重衰减

| 场景 | 推荐 weight_decay |
|------|--------------------|
| 小数据集 | 1e-3 - 1e-4 |
| 中等数据集 | 1e-4 - 1e-5 |
| 大数据集 | 1e-5 - 0 |

## 训练监控

### 关键指标

1. **训练损失**: 应持续下降
2. **验证损失**: 应下降后趋于平稳
3. **验证准确率**: 应上升后趋于平稳
4. **学习率**: 按预期调度

### TensorBoard 可视化

启动 TensorBoard 监控训练过程：

```bash
tensorboard --logdir output/
```

## 常见问题及解决方案

### 训练不收敛

可能原因：
- 学习率过高或过低
- 数据未正确归一化
- 模型架构选择不当

解决方案：
1. 降低/提高学习率
2. 检查数据预处理
3. 尝试不同的模型

### 过拟合

症状：
- 训练损失持续下降
- 验证损失开始上升
- 训练准确率远高于验证准确率

解决方案：
1. 增加 dropout_p
2. 增加 weight_decay
3. 减少模型复杂度
4. 增加数据增强
5. 提前停止训练

### 欠拟合

症状：
- 训练损失和验证损失都很高
- 训练准确率和验证准确率都很低

解决方案：
1. 增加模型复杂度
2. 减少正则化
3. 增加训练轮数
4. 检查数据质量

## 调优流程建议

1. **初始设置**: 使用默认参数进行一次完整训练
2. **观察结果**: 分析训练曲线和指标
3. **识别问题**: 确定是过拟合、欠拟合还是其他问题
4. **调整参数**: 根据问题针对性调整 1-2 个参数
5. **重新训练**: 使用新参数重新训练
6. **对比结果**: 与之前的结果对比
7. **迭代优化**: 重复上述步骤直到满意
